Task: Identificar las clases y sus metodos
Se identifican las clases que potencialmente tengan comunicación con otras, ademas se toma en cuenta sus metodos.
Purpose
El objetivo de identificar las clases y sus métodos es definir la estructura y el comportamiento del sistema de software
Relationships
RolesPrimary Performer:
  • Arquitecto de software
Additional Performers:
InputsMandatory:
  • Diagrama de clases
    		Optional:
    • None
    Outputs
    • Diagrama de secuencias
      Main Description
      Para identificar las clases, es necesario analizar los requisitos del sistema y las funcionalidades que debe proporcionar. Se deben identificar los objetos o conceptos clave en el dominio del problema y agruparlos en clases lógicas y coherentes
      Key Considerations
       Identificar, clases, métodos, estructura, comportamiento, requisitos, funcionalidades, operaciones, dominio del problema, objetos, coherencia, cohesión.
