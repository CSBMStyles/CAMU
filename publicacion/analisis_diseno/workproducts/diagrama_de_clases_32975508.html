Artifact: Diagrama de clases
Es un modelo que establece las entidades (y sus atributos) y actores implicados del sistemas
Purpose
El objetivo de un diagrama de clases es visualizar y representar la estructura estática de un sistema de software, mostrando las clases, sus atributos y relaciones, así como los métodos y comportamientos asociados. El diagrama de clases permite comprender la organización y las interacciones entre las diferentes partes del sistema, facilitando el diseño, la comunicación y la implementación efectiva del software.

Description
Main Description Un diagrama de clases es una representación gráfica que muestra las clases y sus relaciones en un sistema de software. Se utiliza en la etapa de diseño para modelar la estructura estática de un sistema, identificar las clases principales, definir sus atributos y representar las relaciones entre ellas.
