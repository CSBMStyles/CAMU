Artifact: Documento de requisitos funcionales
Documento que esclarece las necesidades que tiene el cliente
Purpose
Establecer una comprensión clara y mutua entre las partes interesadas y el equipo de desarrollo sobre qué funcionalidades debe tener el sistema y cómo estas funcionalidades deben comportarse.
Relationships
RolesResponsible:
  • Analista de negocio
Modified By:
  • Analista de negocio
TasksInput To:
  • Analizar los requerimientos del proyecto
  • Desarrollar codigo
  • Documentar la estructura
  • Identificar las funcionalidades
  • Identificar los requisitos del usuario
  • Organizar en modulos
  • Priorizar las funcionalidades
Output From:
  • Documentacion de los requisitos funcionales
  • Recopilacion y analisis del negocio
Description
Main Description Documento formal que describe de manera detallada las funcionalidades y capacidades que se esperan de un sistema de software, es creado durante la fase de análisis de requisitos y sirve como una guía para el diseño, desarrollo e implementación del sistema.
