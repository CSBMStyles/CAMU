Task: Crear diagrama
Por ultimo, con la información obtenida y estructurada se crea el diagrma de casos de uso del sistema
Purpose
Representar visualmente los diferentes casos de uso y relacionarlos en un diagrama.
Relationships
RolesPrimary Performer:
  • Arquitecto de software
Additional Performers:
InputsMandatory:
  • Lista actores de sistema
    		Optional:
    • None
    Outputs
    • Diagrama de casos de uso de sistema
      Main Description
      Estos casos de uso se representan en el diagrama de casos de uso de sistema como cajas, con líneas que indican las relaciones entre ellos, también se pueden incluir actores, que representan a los usuarios o sistemas externos que interactúan con el sistema.
      Key Considerations

      Es importante que el diagrama de casos de uso de sistema sea claro y fácil de entender, para que pueda ser utilizado como una herramienta de comunicación efectiva entre los desarrolladores, los usuarios y otros interesados en el proyecto.

      El diagrama debe ser lo suficientemente detallado como para mostrar todas las funcionalidades y acciones que el sistema debe realizar, pero también debe ser lo suficientemente simple
