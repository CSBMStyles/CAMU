Task: Identificar los casos de uso
Identificar y definir los diferentes casos de uso que el sistema.
Purpose
Identificar y definir los diferentes casos de uso que el sistema para cumplir con los requisitos del diagrama.
Relationships
RolesPrimary Performer:
  • Arquitecto de software
Additional Performers:
InputsMandatory:
  • Lista actores de sistema
    		Optional:
    • None
    Outputs
    • Diagrama de casos de uso de sistema
      Main Description

      Identificar los diferentes actores que interactúan con el sistema y definir sus roles y responsabilidades, luego, se deben identificar los diferentes casos de uso que el sistema debe ser capaz de realizar para satisfacer las necesidades de los actores identificados.

      Estos casos de uso deben ser definidos de manera clara y concisa, y deben estar relacionados entre sí para representar de manera efectiva el flujo de trabajo del sistema.

      Key Considerations
      Es importante tener en cuenta que los casos de uso deben ser definidos desde la perspectiva del usuario y no desde la perspectiva técnica del sistema.
