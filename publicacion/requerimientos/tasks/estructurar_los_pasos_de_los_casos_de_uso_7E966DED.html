Task: Estructurar los pasos de los casos de uso
Organizar los diferentes pasos de los casos de uso en una secuencia lógica y coherente
Purpose
Organizar los diferentes pasos de los casos de uso identificados, de manera que se puedan representar de manera clara y concisa en el diagrama de casos de uso de sistema.
Relationships
RolesPrimary Performer:
  • Arquitecto de software
Additional Performers:
InputsMandatory:
  • Lista actores de sistema
    		Optional:
    • None
    Outputs
    • Diagrama de casos de uso de sistema
      Main Description
      Estructurar los diferentes pasos de los casos de uso, de manera que representen de manera efectiva el flujo de trabajo del sistema, también es importante asegurarse de que los pasos estén relacionados entre sí de manera clara y concisa, y que se hayan identificado todas las excepciones y condiciones especiales que puedan surgir durante la ejecución del caso de uso.
       
      Una vez que se han estructurado los pasos de los casos de uso, se pueden utilizar para diseñar el diagrama de casos de uso de sistema
