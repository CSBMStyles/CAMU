Task: realizar push o guardar
Por ultimo, una vez aprobado el desarrollo se realiza un backup del codigo.
Purpose
 El objetivo de esta tarea es guardar o subir los cambios realizados en el código fuente a un repositorio centralizado o a un sistema de control de versiones
Relationships
RolesPrimary Performer:
  • Desarrollador Backend
Additional Performers:
InputsMandatory:
  • Codigo del proyecto backend
    		Optional:
    • None
    Outputs
    • Informe de validaciones
      Main Description
       Los desarrolladores backend utilizan herramientas de control de versiones como Git para realizar el push o guardar los cambios. Esto implica seleccionar los archivos modificados, prepararlos para la confirmación y enviarlos al repositorio centralizado.
      Key Considerations
      realizar push, guardar, repositorio centralizado, sistema de control de versiones, Git.
